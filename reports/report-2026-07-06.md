### 🔴 CRITICAL — `CVE-2026-48558`

**SimpleHelp Authentication Bypass Vulnerability** — SimpleHelp  / SimpleHelp

📅 Added to KEV: 2026-06-29 · ⏳ CISA due date: 2026-07-02 · 💀 Ransomware: Unknown

#### Executive summary

SimpleHelp software has a critical security flaw in its OIDC authentication process, allowing unauthorized access. An attacker can bypass the login system, including multi-factor authentication in some cases, by submitting a fabricated identity token without a valid cryptographic signature. This enables a remote, unauthenticated attacker to gain full control of a technician session.

#### Affected technologies

SimpleHelp

#### Recommended actions

  1. Immediately apply all vendor-recommended mitigations for SimpleHelp to address the OIDC authentication bypass vulnerability. Prioritize this action in accordance with CISA's BOD 26-04 guidance.
  2. If vendor mitigations are not available, discontinue the use of SimpleHelp.
  3. Evaluate all SimpleHelp assets for internet exposure and ensure adherence to BOD 26-04 patching guidelines.
  4. Implement forensic triage requirements as outlined by CISA for any potentially affected systems.

#### Rationale

_This vulnerability is classified as critical because it allows a remote, unauthenticated attacker to achieve full authentication, potentially bypassing multi-factor authentication, by forging identity tokens. This leads to a complete compromise of a technician session, granting high privileges without any prior authentication, making it extremely severe._
