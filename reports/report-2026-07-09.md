### 🟠 HIGH — `CVE-2026-55255`

**Langflow Authorization Bypass Through User-Controlled Key Vulnerability** — Langflow / Langflow

📅 Added to KEV: 2026-07-07 · ⏳ CISA due date: 2026-07-10 · 💀 Ransomware: Unknown

#### Executive summary

An authorization bypass vulnerability has been identified in Langflow that allows authenticated attackers to execute any data flow belonging to another user. This is achieved by manipulating the flow ID in a request, enabling unauthorized access to and execution of other users' processes. Organizations using Langflow should take immediate action to address this critical security flaw.

#### Affected technologies

Langflow

#### Recommended actions

  1. Apply mitigations in accordance with vendor instructions as soon as they are available.
  2. Evaluate the internet exposure of all Langflow instances within the organization.
  3. Ensure adherence to CISA's BOD 26-04 patching guidelines for all affected assets.
  4. If mitigations are unavailable or cannot be applied promptly, discontinue use of the Langflow product to prevent exploitation.
  5. Implement monitoring for unusual activity related to flow execution in Langflow environments.

#### Rationale

_The severity is classified as high because this is an authorization bypass vulnerability actively exploited in the wild (as it's in CISA KEV) which allows an authenticated attacker to execute arbitrary flows belonging to other users. This could lead to unauthorized data access, manipulation, or denial of service, depending on the nature of the flows. The requirement for prior authentication slightly reduces it from critical, but the impact of full authorization bypass for other users' resources is significant._
