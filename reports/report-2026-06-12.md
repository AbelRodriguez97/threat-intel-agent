Exploitation is possible if the appliance is unmanaged and externally reachable.

Ivanti Sentry, MobileIron Sentry

  1. Apply mitigations in accordance with vendor instructions, ensuring compliance with CISA's BOD 26-04 Prioritizing Security Updates Based on Risk guidance.
  2. Follow CISA's 'Forensics Triage Requirements'.
  3. Evaluate each asset's internet exposure and ensure adherence to BOD 26-04 patching guidelines.
  4. For cloud services, follow applicable BOD 26-04 guidance or discontinue use if mitigations are unavailable.
  5. If the Sentry appliance is in an unmanaged state with externally reachable endpoints, implement mTLS with EPMM or restricted HTTPS access through Neurons for MDM to make interfaces inaccessible to external actors.

#### Rationale

_This vulnerability allows for root-level remote code execution by a remote, unauthenticated attacker. The severity is critical due to the complete compromise of the system without requiring any prior access or authentication, and its inclusion in CISA's KEV catalog confirms active exploitation._

### 🔴 CRITICAL — `CVE-2026-11645`

**Google Chromium V8 Out-of-Bounds Read and Write Vulnerability** — Google / Chromium V8

📅 Added to KEV: 2026-06-09 · ⏳ CISA due date: 2026-06-23 · 💀 Ransomware: Unknown

#### Executive summary

A critical vulnerability in the Chromium V8 engine allows attackers to execute unauthorized code simply by having a user visit a malicious website. This flaw affects popular browsers like Google Chrome, Microsoft Edge, and Opera, enabling potential compromise even within browser security sandboxes. Organizations should prioritize patching to prevent remote code execution.

#### Affected technologies

Google Chromium V8, Google Chrome, Microsoft Edge, Opera, Chromium-based web browsers

#### Recommended actions

  1. Immediately apply all available security updates and vendor-provided mitigations for Google Chromium V8 and any Chromium-based web browsers in use.
  2. For cloud services utilizing Chromium, adhere strictly to CISA's BOD 22-01 guidance.
  3. If vendor mitigations are not available or cannot be applied promptly, consider discontinuing the use of affected products until a secure update is deployed.

#### Rationale

_This vulnerability is classified as critical due to its potential for remote code execution (RCE) through a widely accessible attack vector (crafted HTML page). The flaw resides in the Chromium V8 engine, impacting numerous popular web browsers, and is actively exploited as confirmed by its inclusion in the CISA KEV catalog. The out-of-bounds read and write capabilities present a severe risk of memory corruption and system compromise._
